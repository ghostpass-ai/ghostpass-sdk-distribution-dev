## 1. 에러 코드 체계

### 1.1 에러 코드 형식

모든 에러는 `GPxxx` 형식의 코드를 가지며, 로그에는 내부 추적 코드가 함께 출력됩니다.

```
GP103(IN-107)
─┬── ──┬────
 │     └─ Internal 코드 (디버깅·기술 지원용)
 └─ External 코드 (파트너사 분기 처리용)
```

- **External 코드 (`GPxxx`)** — 파트너사에서 분기 처리에 사용합니다.
- **Internal 코드 (`IN-xxxx`)** — GhostPass 기술 지원 문의 시 함께 전달해 주세요.

### 1.2 에러 코드 대역

User App SDK와 Kiosk SDK는 **독립된 SDK**이므로 동일한 GP 코드 번호가 존재할 수 있습니다.  
반드시 **SDK 유형을 구분**하여 참조하세요.

## 2. User App SDK

### 2.1 GP0xx — 백그라운드 서비스 (`onServiceError`)

비콘 탐지 및 인증 과정에서 SDK가 자율적으로 발생시키는 에러입니다.  
`NotificationCenter` (iOS) 또는 콜백 리스너 (Android)를 통해 수신합니다.

| 에러 코드 | 설명 | 권장 대응 |
|-----------|------|----------|
| GP001 | 블루투스를 활성화해주세요. | 블루투스 ON 안내 |
| GP002 | 블루투스 권한이 필요합니다. | 설정 화면으로 이동 안내 |
| GP003 | 위치 서비스 권한이 필요합니다. | 설정 화면으로 이동 안내 |
| GP004 | 위치 서비스 권한이 부족합니다. | 설정에서 '항상 허용'으로 변경 안내 |
| GP005 | 인증 정보를 확인할 수 없습니다. | initialize() 재시도 |
| GP006 | 네트워크 연결을 확인해주세요. | 네트워크 상태 확인 안내 |

### 2.2 GP1xx — `initialize(apiKey:)`

| 에러 코드 | 설명 | 권장 대응 |
|-----------|------|----------|
| GP101 | 초기화를 실패했습니다. | 일정 시간 후 재호출 |
| GP102 | SDK 버전이 최신이 아닙니다. | SDK 업데이트 필요 |
| GP103 | 요청 파라미터가 올바르지 않습니다. | API Key 확인 후 재호출 |

### 2.3 GP2xx — `registerBioData(imageBytes:)` / `removeBioData()`

| 에러 코드 | 설명 | 권장 대응 |
|-----------|------|----------|
| GP201 | 생체 데이터 등록에 실패했습니다. | 생체 데이터 등록 재시도 |
| GP202 | 생체 데이터 삭제에 실패했습니다. | 생체 데이터 삭제 재시도 |
| GP203 | 입력 이미지가 비어 있거나 손상되었습니다. | 이미지 데이터를 확인한 뒤 다시 호출 |
| GP204 | 라이브니스 검증을 통과하지 못했습니다. | 실제 이미지 데이터로 시도 |

### 2.4 GP3xx — `reset()`

| 에러 코드 | 설명 | 권장 대응 |
|-----------|------|----------|
| GP301 | 초기 상태 복구에 실패했습니다. | 앱 재시작 후 `initialize()` 재호출 |

### 2.5 GP4xx — `delegateAuthSession(session:)`

| 에러 코드 | 설명 | 권장 대응 |
|-----------|------|----------|
| GP401 | 핸즈프리 인증 처리에 실패했습니다. | Push payload 및 유저앱 내 생체 데이터 저장 여부 확인 후 재시도 |

### 2.6 GP9xx — 서버

| 에러 코드 | 설명 | 권장 대응 |
|-----------|------|----------|
| GP901 | SDK 서비스를 찾을 수 없습니다. | GhostPass 담당자에게 서비스 등록 확인 |
| GP902 | 비활성화된 SDK 서비스입니다. | GhostPass 담당자에게 서비스 활성화 요청 |
| GP903 | 이미 사용 중인 서비스 코드입니다. | GhostPass 담당자에게 문의 |
| GP904 | 이미 사용 중인 API Key입니다. | GhostPass 담당자에게 문의 |
| GP905 | 서버의 응답이 비정상적입니다. | 일정 시간 후 재호출 |
| GP906 | 내부 처리 중 오류가 발생했습니다. | initialize() 재시도 |

## 3. Kiosk SDK

### 3.1 GP0xx — `initialize()`

| 에러 코드 | 설명 | 권장 대응 |
|-----------|------|----------|
| GP001 | 파라미터 누락 또는 형식 오류 | 입력값 수정 후 재호출 |
| GP002 | 이미 등록된 키오스크 | 관리자 확인 (무시 가능) |
| GP003 | 미등록 키오스크 ID | 관리자에게 등록 요청 |
| GP004 | 비활성 상태의 키오스크 | 관리자에게 활성화 요청 |
| GP005 | 키오스크 ID 규격 불일치 | ID 수정 후 재호출 |
| GP006 | 내부 엔진 및 통신 초기화 실패 | 5초 후 재호출 (Retry) |
| GP007 | 블루투스 OFF | 블루투스 ON |
| GP008 | Manifest 블루투스 권한 없음 | `BLUETOOTH_ADVERTISE`, `BLUETOOTH` 권한 선언 |
| GP009 | 블루투스 비콘 송출 미지원 기기 | 대응 불가 (기기 교체 필요) |

### 3.2 GP9xx — 공통 (네트워크 · 서버)

| 에러 코드 | 설명 | 권장 대응 |
|-----------|------|----------|
| GP901 | 기기 네트워크 연결 불가 | 네트워크 복구 후 재호출 |
| GP902 | 서버 비정상 응답 (5xx 등) | 5초 후 재호출 |
| GP903 | API Key 인증 실패 (401) | **Key 확인 필수 (재시도 불가)** |
| GP904 | SDK 내부 로직 오류 | `causeCode` 확인 후 재호출 |
| GP905 | 리셋 실패 | `initialize()` 한 번 더 호출 후 앱 재시작 |
| GP999 | 예상치 못한 오류 | `causeCode` 확인 후 기술 지원 문의 |

### 3.3 `detection()` — 결과 타입

`detection()`은 예외를 던지지 않고 `GoPassAuthResult` 객체를 반환합니다.

#### A. PrecheckGuide (얼굴 검증 및 상태 가이드)

카메라 프레임의 얼굴 상태에 따른 UI 가이드입니다. **에러가 아니며**, 다음 프레임에서 자동으로 재시도됩니다.

| FailReason | UI 가이드 메시지 (예시) | 권장 대응 |
|------------|----------------------|----------|
| `NO_FACE` | 얼굴이 인식되지 않았어요. 카메라를 바라봐주세요 | 다음 프레임에서 재호출 |
| `INVALID_POSE_ROLL` | 고개를 삐딱하지 않게 똑바로 세워주세요 | 다음 프레임에서 재호출 |
| `INVALID_POSE_PITCH` | 고개를 숙이거나 들지 말고 정면을 봐주세요 | 다음 프레임에서 재호출 |
| `INVALID_POSE_YAW` | 옆을 보지 말고 카메라 렌즈를 봐주세요 | 다음 프레임에서 재호출 |
| `INVALID_POSITION` | 얼굴을 화면 중앙에 위치시켜주세요 | 다음 프레임에서 재호출 |
| `INVALID_SIZE_MAX` | 화면에서 한 걸음 뒤로 물러나주세요 | 다음 프레임에서 재호출 |
| `INVALID_SIZE_MIN` | 카메라 쪽으로 한 걸음만 다가와주세요 | 다음 프레임에서 재호출 |
| `NO_SESSION` | 인증 대기자가 존재하지 않습니다 | 대기 화면 표시 (SDK 자동 복구) |
| `LIVENESS_NOT_CONFIRMED` | 실제 사람인지 확인 중입니다. 잠시 멈춰주세요 | 다음 프레임에서 재호출 |
| `LIVENESS_TX_FAILED` | 인증에 실패하였습니다. 다시 시도해주세요 | 다음 프레임에서 재호출 |
| `ALREADY_IN_PROGRESS` | 인증을 진행하고 있습니다... | 다음 프레임에서 재호출 |

#### B. ServerAuthFailed (서버 인증 단계 실패)

| 에러 코드 | 설명 | 권장 대응 |
|-----------|------|----------|
| GP101 | 인증 시간 초과 (Timeout) | 메시지 표시 후 다음 프레임 재호출 |
| GP102 | 유저 앱에서 인증 거절 | 메시지 표시 후 다음 프레임 재호출 |
| GP103 | 서버 인증 과정 중 예외 발생 | 메시지 표시 후 다음 프레임 재호출 |
